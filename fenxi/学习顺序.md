# Vulkan 示例学习顺序指南

本文档为 Vulkan 示例工程提供了从入门到高级的推荐学习顺序，帮助您系统地掌握 Vulkan 图形编程。

## 📚 学习路径概览

```
基础入门 → 纹理与渲染 → 高级渲染技术 → 性能优化 → 现代渲染技术 → 扩展功能
```

---

## 🎯 第一阶段：基础入门（必须掌握）

这些示例是学习 Vulkan 的基础，建议按顺序学习。

### 1. **triangle** - 基础三角形渲染
- **难度**: ⭐
- **重要性**: ⭐⭐⭐⭐⭐
- **学习内容**: 
  - Vulkan 初始化流程
  - 实例、设备、交换链创建
  - 命令缓冲区、同步对象
  - 顶点缓冲区和索引缓冲区
- **为什么先学**: 这是最基础的示例，展示了 Vulkan 的完整渲染流程，没有使用任何辅助函数
- **预计时间**: 2-3 天

### 2. **trianglevulkan13** - Vulkan 1.3 版本三角形
- **难度**: ⭐
- **重要性**: ⭐⭐⭐⭐
- **学习内容**:
  - Vulkan 1.3 新特性
  - 动态渲染（Dynamic Rendering）
  - 简化的 API 使用
- **为什么学**: 了解现代 Vulkan API 的简化方式
- **预计时间**: 1 天

### 3. **pipelines** - 管线状态对象
- **难度**: ⭐⭐
- **重要性**: ⭐⭐⭐⭐⭐
- **学习内容**:
  - 图形管线创建
  - 管线状态对象（PSO）
  - 管线派生
  - 多种渲染模式（Phong、线框、卡通）
- **为什么学**: 理解 Vulkan 的核心概念——管线状态对象
- **预计时间**: 1-2 天

### 4. **descriptorsets** - 描述符集
- **难度**: ⭐⭐
- **重要性**: ⭐⭐⭐⭐⭐
- **学习内容**:
  - 描述符集布局
  - 描述符池和描述符集
  - Uniform 缓冲区绑定
  - 多对象渲染
- **为什么学**: 描述符是 Vulkan 中传递数据到着色器的核心机制
- **预计时间**: 2 天

### 5. **pushconstants** - 推送常量
- **难度**: ⭐⭐
- **重要性**: ⭐⭐⭐⭐
- **学习内容**:
  - 推送常量的使用
  - 小数据快速传递
  - 避免描述符集开销
- **为什么学**: 学习高效传递小数据的方法
- **预计时间**: 1 天

### 6. **specializationconstants** - 特化常量
- **难度**: ⭐⭐
- **重要性**: ⭐⭐⭐
- **学习内容**:
  - SPIR-V 特化常量
  - 从单一着色器创建多个管线
  - 编译时优化
- **为什么学**: 了解如何优化着色器编译
- **预计时间**: 1 天

## 📝 学习建议

### 时间规划
- **第一阶段（基础）**: 1-2 周
- **第二阶段（纹理）**: 1-2 周
- **第三阶段（高级渲染）**: 2-3 周
- **第四阶段（性能优化）**: 1-2 周
- **第五阶段（现代渲染）**: 3-4 周
- **第六阶段（计算着色器）**: 2-3 周
- **第七阶段（几何/曲面细分）**: 2-3 周
- **第八阶段（光线追踪）**: 3-4 周（需要 RTX 硬件）
- **后续阶段**: 根据兴趣选择

### 学习方法
1. **先理解概念**: 阅读代码前先理解技术原理
2. **动手实践**: 修改代码，观察效果变化
3. **调试工具**: 使用 RenderDoc 等工具分析渲染流程
4. **循序渐进**: 不要跳过基础示例
5. **做笔记**: 记录关键概念和 API 用法

### 必备工具
- **RenderDoc**: 图形调试器
- **Vulkan SDK**: 包含验证层和工具
- **Visual Studio / IDE**: 代码编辑和调试
- **GPU 厂商工具**: NVIDIA Nsight、AMD Radeon GPU Profiler

### 常见问题
1. **验证层错误**: 仔细阅读错误信息，通常是资源生命周期问题
2. **同步问题**: 理解内存屏障和同步原语
3. **性能问题**: 使用性能分析工具定位瓶颈
4. **扩展支持**: 检查设备是否支持所需扩展
